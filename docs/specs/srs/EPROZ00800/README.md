# EPROZ00800 — 退補件 / 修正項目（Revised Item）｜開發速覽

## 這頁在幹嘛（一句）
承辦人對**退補（修正）案件**勾選「這次要修哪些項目」（共 14 個修正項 + 一段原因說明），存檔時系統會依勾選變化**連動清掉/還原**相關的保證人、擔保品、放款條件等資料，並更新案件完成狀態。本頁是**共用模組**，但**只在展期 / 展變兩種案型才顯示與啟用**（`R1`）。

## 怎麼運作〔此段隨頁型態而異，非通用範本〕
- **載入（init-query）**：進頁先由後端取案號、案型、可編輯性、CU/有無擔保等屬性，再讀出該案目前已存的 14 個修正項與原因說明帶回畫面；沒存過則項目留空（`R1`/`R2`）。
- **編輯**：使用者勾選/取消修正項、填原因說明。部分項目由案型/案件屬性**強制鎖定**——展期案強制勾首項並鎖住、非展期則強制不勾；企業未擔保案強制不勾擔保項並鎖住（`R5`/`R6`）。
- **存檔前警告**：若勾選和原本不同，前端要先警告「改/取消修正項會清掉相關已填資料」；只有第一次存（先前全空）才略過（`R8`）。
- **存檔（save/execute）**：單一交易內——比對 DB 現況 vs 送來的勾選（以**後端 DB 比對**為準、非前端旗標），先**整列替換**修正項，再依勾選變化跑保證人/擔保品/放款條件/費用的**清除或還原**副作用，最後更新案件 checkpoint；任一步失敗整筆 rollback（`R9`–`R14`）。
- **存完刷新**：前端**另呼**分頁授權/狀態 endpoint 重抓分頁狀態——存檔回應**不**負責帶回權威的選單/狀態 key（`R14`）。

## 規則速覽（白話；精確值見對應 `Rn`）
**主流程（依執行序）**
1. **顯示判定** `R1`：只有展期 / 展變案才顯示與啟用；其他案型前端隱藏、後端直呼查詢/存檔**在讀資料/動 DB 前擋下**。案型一律由**後端**判（不信前端）。
2. **載入** `R2`：查詢帶回案型、客戶/擔保屬性、現有 14 修正項與原因；另回**後端算好的「企業未擔保」旗標**（前端不可只憑有無擔保自己推）。
3. **顯示名稱** `R3`：14 個修正項依正式選項名顯示；選項維護不在本頁。
4. **驗證** `R4`：至少勾一項；各項只能勾/不勾；原因說明必填、去空白、上限 3000 字（注意 DB 欄更寬，契約不得放寬）。展期 / 展變**共用同一套驗證**。
5. **鎖定規則** `R5`/`R6`：展期強制勾首項並鎖、非展期強制不勾並鎖；企業未擔保強制不勾擔保項並鎖。
6. **存檔副作用** `R9`–`R13`：以後端 DB 比對為權威觸發；保證人項 Y→N 清除並由參考案還原 + 更新借款人旗標；擔保項 Y→N 清除擔保相關資料並還原 + 更新擔保提供旗標；放款條件項整組開/關切換時清除明細並標記下游頁重跑、單項 Y→N 則還原原值；展變案費用項 N→Y 清費用列；末兩項僅存不連動。
7. **checkpoint** `R14`：依**檢核家族**（客戶/擔保屬性）寫對應的新版 checkpoint 表 + 本頁自身 key + 已核准的下游重跑 key；下游 0150 只在實體有該 key 的家族更新、無 key 的家族不得自創；0173 屬摘要/唯讀顯示、非本頁 checkpoint 目標。

**橫切（每階段都適用）**
- **錯誤/rollback** `R9`/`R15`：副作用/checkpoint 任一失敗整筆 rollback；錯誤碼以 PRD 那組為準（見 spec `R15`），欄位驗證細節只放錯誤 envelope 的 data。
- **授權** `R16`（mutating）：初始化/查詢/存檔都要後端授權；**只設 API 授權不夠**，存檔要在動任何資料前用後端 page/edit 授權擋直呼。
- **稽核** `R17`：存檔記副作用類別/列數/結果/耗時摘要，**不**記完整原因說明/借款人姓名/敏感內容。
- **邊界/NFR** `R18`：載入/查詢 ≤3 秒、存檔 ≤5 秒、交易逾時 ≤30 秒；log 不得外洩 SQL/stack/完整個資/完整 payload。

## NFR（精確見 spec `## NFR`）
交易（修正項替換 + 副作用 + checkpoint + 稽核**同生同滅**，僅失敗稽核例外）／安全（mutating 後端授權、API 授權單獨不足）／精度（旗標 1 byte、原因說明輸入上限 < DB 欄寬、放款金額/利率等還原須保留 DB 精度）／隱私／觀測（留 requestId + 案號可追）。

## 資料表（確切表名/欄位/PK 見 `schema.sql`）
主寫入＝修正項主表（整列替換）＋修正項明細表（放款條件還原用）；來源／判定＝案件摘要表（案型、客戶/擔保屬性、參考案號）；副作用波及＝保證人、擔保品、放款條件/費用、借款人旗標等多張活表；另寫＝案件 checkpoint 表（新版四家族；舊 RC 家族表已停用）。

## ⚠️ 雷區（依 RD 接手順序）
**動工前必知**
- **顯示 = 後端判**：本頁雖叫「共用」，但只在展期 / 展變顯示啟用；非此案型要在讀資料/動 DB 前擋直呼，別只靠前端隱藏。
- **副作用觸發以 DB 比對為權威**：用後端讀現況 vs 送來狀態比對，**不可**拿前端「有沒有變」旗標當觸發；該旗標只代表「警告有跳」。
- **整筆交易**：替換 + 副作用 + checkpoint + 稽核全有無；任一失敗 rollback。

**實作踩坑**
- **放款條件整組刪除條件**：現碼對「整組開/關切換」的判定與 legacy 目標**有落差**（implementation delta），須修正或經 owner 接受才可 closeout（見 spec `R13`）。
- **企業未擔保旗標**：由後端用客戶屬性 + 擔保屬性**兩者**合判，前端不可只看擔保屬性自推（`R6`）。
- **原因說明欄寬**：DB 實體欄比契約輸入上限寬，契約不得靜默放寬（`R4`/`R10`）。
- **checkpoint key 不對稱**：只有特定家族有下游 0150 key，其餘家族不得硬塞；0173 不是本頁的 checkpoint 目標（`R14`）。
- **執行請求形狀已定版**：以 spec `R11` 為準，舊式 checkPointMap 不是 client 可帶的權威輸入，checkpoint/選單由後端推導。

**維護註記**
- 本包所有 @PENDING（RP1–RP11、BP1–BP5）皆 ✅ 已關；授權/page-column backfill 已套用、closeout 綠（實作完成軸仍由 DoD 閘門牆裁）。
- 末兩個修正項目前僅存不連動，若日後有 owner 核可的新證據才加副作用（`R13`）。
