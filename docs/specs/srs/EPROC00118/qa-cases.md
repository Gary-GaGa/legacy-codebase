## Execution Details

Every error case must assert `code`, `message`, and `data` in the response body. The actual HTTP status is recorded against the platform exception mapping; until `TBD-ERR-STATUS` is adjudicated, the status number is evidence, while missing error body, unexpected success, or DB mutation is a failed case.

## DB Verification Seeds
- `TB_CORP_SCRCARD.APPLICATION_NO='xxx'` 且 AO/CR 欄位各一組，供 QA-001/003。
- `TB_SCORE_CARD_PARAM_DETAIL` 至少包含 `B_V1`~`B_V22` 與 `COR_RISK_LV` 的有效 `ST_DATE` 版本，供 QA-002/010。
- `TB_LON_SUMMARY_INFO.APPLICATION_NO='xxx'`，`LON_ATTRIBUTE`/`SECURE_ATTRIBUTE` 分別覆蓋 CS/CU，供 QA-017。
- `TB_CHECK_POINTS_CS`、`TB_CHECK_POINTS_CU` 各有 `EPROC00118` 初始 `'N'` 或 `'Y'` rows，供 QA-007/017。
