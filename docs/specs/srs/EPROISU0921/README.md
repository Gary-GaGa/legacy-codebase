## 這頁在幹嘛（一句）
CAD Maker 為**個金撥貸案**輸入撥貸資料（金額、撥款帳、律師費、擔保品、加購不動產等），檢核主借/共同借款人身分，存草稿或 **Finished 定版**；Finished 後此案的「完成旗標」才放行上一層 Summary 頁。**這是 T1 金額/checkpoint/授權頁，後端為一切權威。**

## 怎麼運作〔此段隨頁型態而異，非通用範本〕
1. **載入選項**（`R1`）：進頁先呼 select endpoint 拿下拉/靜態值、可見律師事務所、address 版本日與省份清單。此 endpoint 是會寫資料的 POST（含一個保留的初始化副作用），須交易安全。
2. **查頁狀態**（`R2`）：呼 info endpoint 拿 summary、撥貸日期、擔保品/加購清單、共同借款人清單、主/共借檢核狀態、帳號候選與**完成旗標**。若尚無撥貸日期資料，由核准的貸款條件預填幣別/金額，但**不標完成**。
   - 此處的 address 版本日（`R1` 載入的那個值）即各層地址查詢的依據。
3. **輸入撥貸資料 + 身分檢核**：
   - 主借款人檢核（`R4`）：比對姓名/生日/性別/產業別等，依**產品碼→產業/行業對照**（封閉集、未列即失敗）判定；通過需「今日」檢核且比對成功。
   - 共同借款人檢核（`R5`/`R8`）：逐筆比對，全部過才算過；**無共同借款人＝視為通過、不擋 Finished**（A-4 舊基準）。
   - 上游服務逾時/掛掉**不可偽裝成「檢核未過」**，要回真正的錯誤。
4. **存草稿 / Finished**（`R6`/`R7`/`R10`）：
   - 草稿：存目前輸入、不設完成旗標。
   - Finished：先過所有必填/金額/地址/律師費/檢核/角色驗證，再**原子**寫入撥貸主檔與擔保品檔，成功才設完成旗標。
5. **Return**（`R9`）：把案件退回前一關，原子地寫退件歷程、改合約狀態、清掉本流程產生的暫存資料（**只清 DB，不刪實體檔**）。

## 規則速覽（白話；精確值見對應 `Rn`）
**主流程（依執行序）**
1. **載入選項** `R1`：律師事務所只露「可顯示」者；address 版本日沿用舊系統的申請日衍生值。
2. **查頁狀態** `R2`：完成旗標為對外旗標，**只有它為 Y** 上層 Summary 才放行（`R12`）。
3. **主借檢核** `R4`：產品碼→產業/行業是**封閉對照**，未列組合一律失敗、不得預設放行；通過需「今日」檢核。
4. **共借檢核** `R5`/`R8`：全筆過才算過；無共同借款人＝過。識別序號要對得回原列，產業別不符要擋。
5. **存草稿/Finished** `R6`：Finished 前所有驗證 BE 必過才 mutate；金額類（手續費/再融資費）由**後端計算**、非前端值。
6. **完成日期** `R7`：擔保品/加購的完工日對外用月年格式、DB 仍存日期；提供有效值時**不得寫成空**（現碼曾寫空＝要修）。
7. **Return** `R9`：退件清理只動 DB、原子完成。

**橫切（每階段都適用）**
- **冪等（最重要）** `R6`：Finished **不是可重試的金額動作**。重跑 `Finished` 必須先查完成旗標，已完成就**拒絕 + 全程 no-op**（金額/費用/擔保品/暫存都不可再動）。
- **錯誤/上游** `R4`/`R5`：上游逾時/掛掉回真錯，不可偽裝成「檢核未過」。
- **授權** `R3`/`R11`：用頁欄授權決定按鈕/可編欄；後端對所有會寫的 endpoint **再驗角色/狀態**，光有 API 授權列不夠。
- **金額/精度** `R10`：金額遵 DB 精度；USD 用舊系統兩位小數規則（**不可截斷**）、KHR 維持已核准規則；CBC Member 與加購不動產**共用上限 5、但各算各的**，後端要擋超限。
- **隱私/稽核** `R11`：錯誤不得外洩憑證/主機/堆疊/T24 原始資料/個資；mutating 動作要寫稽核歷程。

交易（Finished 與 Return 跨多表原子）／觀測（檢核失敗回結構化訊息、不洩服務細節）／相容（保實體 DB 名與舊完成旗標語意、對外用新契約名）／可測（T1 金額/checkpoint/安全規則皆 BE 驗）。

## 資料表（確切表名/欄位/PK 見 `schema.sql`）
主寫入＝撥貸主檔（含金額、律師費欄、實體完成旗標）＋擔保品/加購不動產檔；身分檢核寫主借/共借 T24 檢核表與主借帳號表；Return 會清這些暫存表並寫退件歷程、改合約狀態。地址相關表以版本日為鍵。

## ⚠️ 雷區（依 RD 接手順序）
**動工前必知**
- **重複 Finished**：T1 金額頁，已完成的案件再 Finished 必須拒絕並全程 no-op，否則會重複撥款副作用（Bible 災難情境）。
- **無共同借款人＝通過**：A-4 舊基準，現碼會回空白＝regression，要修成「視為通過、不擋 Finished」。
- **後端權威**：前端傳的「共借已過」旗標不可信，Finished 前後端要用當前已存檢核狀態重算；角色/狀態後端必擋。

**實作踩坑**
- **rounding**：USD 用舊系統兩位小數格式（現碼用截斷＝要修）；KHR 維持已核准規則。
- **產品碼對照**：產業/行業是封閉集，未列組合一律失敗，別讓現碼的「預設放行」漏網。
- **完工日期**：對外月年、DB 存日期，提供值時不可寫空（現碼寫空＝要修）。
- **共用清單上限**：CBC Member 與加購不動產兩份清單共用同一上限值但**分開計數**、不可相加成一個總額；後端要擋、不可靜默截斷（確切上限值見 spec `R10`）。
- **address 版本日**：沿用舊系統的申請日衍生值，別誤用初始化分支裡的撥貸日。
- **識別序號**：擔保品/加購/共借的序號是穩定識別鍵，save 不可重新編號破壞 T24 比對。
