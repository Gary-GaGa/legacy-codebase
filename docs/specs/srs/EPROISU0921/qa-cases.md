# QA Cases - EPROISU0921 Data Input

| QA ID | covers | Type | Given | When | Then |
|---|---|---|---|---|---|
| QA-001 | R1, R2 | happy | Seed an application with existing `TB_DISBUR_DATE`, collateral, purchased property, borrower check rows, and active law firms. | POST `/epl-sele-isu-data-input`, then POST `/epl-info-isu-data-input`. | Response returns `updDate`, active law firms only, existing Data Input values, `mbCheck`, `coCheck`, collateral lists, and `eprois0921` without missing required render data. |
| QA-002 | R1, R2 | happy | Seed an application without `TB_DISBUR_DATE` but with approved loan-condition/source data. | POST `/epl-info-isu-data-input`. | Response pre-fills disbursement currency, disbursement amount, and borrower T24 number from source data, while `eprois0921` remains blank or `N`. |
| QA-003 | R1 | edge | Seed an application at `CASE_PROGRESS=24` with no current `DISBURSING_DATE` and an `APPLICATION_DATE` that maps to a known address `UPD_DATE`. | POST `/epl-sele-isu-data-input`. | The endpoint explicitly performs the retained disbursing-date initialization side effect transactionally, but returns the application-date-derived address `updDate` used by province/district/commune/village lookups. |
| QA-004 | R2, R12 | error | Seed an application where `TB_DISBUR_DATE.EPORIS_0921` is blank or `N`. | EPROISU0920 Summary gate reads Data Input state through `/epl-info-isu-data-input`. | Summary remains blocked and the UI returns to Data Input; no completion is inferred from partial rows. |
| QA-005 | R3, R11 | error | Authenticate a user without CAD Maker permission or with a case outside editable state. | POST `/epl-save-isu-data-input` with `isFinish=Y`. | BE returns `ACCESS_DENIED` or an invalid-state response before mutating `TB_DISBUR_DATE`, collateral, T24 temp, or history rows. |
| QA-006 | R4 | happy | Seed matching main borrower identity, `BUSINESS_SEC_CODE`, same-currency SSI/account data, and confirmed Product Code mapping (`01`/`02` with Sector/Industry `1001`/`1001`, or `03` with `1002`/`1001`). | POST `/epl-case-isu-data-input-check-mb`. | Response has `mbCheck=Y`, stores today's `CHECK_DATE`, stores `CHECK_SUCCESS=Y`, and returns account options with SSI first. |
| QA-007 | R4 | error | Seed a mismatch in main borrower name, DOB, gender, Sector/Industry, business-section, account currency, or an unlisted product / unlisted Sector-Industry combination. | POST `/epl-case-isu-data-input-check-mb`. | Response has `mbCheck=N`, returns row-level result messages, clears/omits invalid account candidates, and Finished cannot reuse a prior pass. |
| QA-008 | R5, R6 | edge | Seed an application with no co-borrower rows. | POST `/epl-info-isu-data-input`, then POST `/epl-save-isu-data-input` with `isFinish=Y` and a valid main borrower check. | Co-borrower check does not block Finished under the owner-confirmed old baseline; `coCheck` is normalized to `Y`, not blank. |
| QA-009 | R5, R8 | happy | Seed two co-borrowers with stable `DATA_SEQ`; T24/CIF returns matching name, DOB, gender, and `BUSINESS_SEC_CODE` for both rows. | POST `/epl-case-isu-data-input-check-co`. | Response has `coCheck=Y`, and `TB_T24_CO_BORROWER_INFO` rows preserve `DATA_SEQ`, `RECID`, today's `CHECK_DATE`, `BUSINESS_SEC_CODE`, and `CHECK_SUCCESS=Y`. |
| QA-010 | R5 | error | Seed two co-borrowers where one T24 row is stale, missing, failed, or mismatched. | POST `/epl-case-isu-data-input-check-co`, then attempt Finished. | Response has `coCheck=N`; Finished is rejected by BE even if the client submits `coCheck=Y`. |
| QA-011 | R6 | happy | Seed an editable CAD Maker case and submit partial Data Input with `isFinish=N`. | POST `/epl-save-isu-data-input`. | Draft values persist to `TB_DISBUR_DATE`, `TB_DISBUR_COLL`, and `TB_DISBUR_OTHER_COLL`; `EPORIS_0921` remains `N`; `RECEIVED_DATE` is not set as a completion event. |
| QA-012 | R6, R10 | happy | Seed a fully valid editable case with current `mbCheck=Y`, valid co-borrower pass state, required fields, money values, and address lengths. | POST `/epl-save-isu-data-input` with `isFinish=Y`. | Save is atomic, `EPORIS_0921=Y`, `TB_LON_SUMMARY_INFO.RECEIVED_DATE` is updated, and EPROISU0920 Summary gate can proceed. |
| QA-013 | R6 | error | Seed a case with `mbCheck=N`, `coCheck=N`, stale check date, or direct client tampering of `coCheck`. | POST `/epl-save-isu-data-input` with `isFinish=Y`. | BE rejects Finished with the relevant check message and leaves completion flag and received date unchanged. |
| QA-014 | R7 | happy | Seed collateral and purchased-property rows with supplied `estCom` and `otrEstCom` in `MM/YYYY` format. | POST `/epl-save-isu-data-input`, then POST `/epl-info-isu-data-input`. | `TB_DISBUR_COLL.EST_COM_DATE` and `TB_DISBUR_OTHER_COLL.OTHER_EST_COM_DATE` persist month/year and round-trip to `estCom` and `otrEstCom` in `MM/YYYY`; supplied values are not overwritten to null. |
| QA-015 | R7 | edge | Submit applicable collateral and purchased-property rows without completion dates. | POST `/epl-save-isu-data-input`. | Date columns remain null only for missing/non-applicable values; existing supplied values are not overwritten to null by unrelated saves. |
| QA-016 | R8 | happy | Seed existing collateral and purchased-property rows with non-contiguous sequence numbers. | POST `/epl-save-isu-data-input` with updates to the same rows. | Existing `COLL_DATA_SEQ` and `OTHER_COLL_DATA_SEQ` identities are preserved; downstream rows still reference the same sequence values. |
| QA-017 | R8 | error | Seed a co-borrower whose CIF source `BUSINESS_SEC_CODE` differs from the application co-borrower source row. | POST `/epl-case-isu-data-input-check-co`. | Response has `coCheck=N`, row result explains business-section mismatch, and stored T24 temp row records the failed result. |
| QA-018 | R9 | happy | Seed an editable case with Data Input rows, T24 temp rows, `TB_MAIN_BORROWER_ACC`, `TB_CONTR_AUTO_FILE_PTH`, FN loan-condition fee/detail rows, `TB_CONTR_DATA`, and a valid return actor. | POST `/epl-retu-isu-data-input`. | Case returns to the expected prior workflow state, `TB_APP_HISTORY` is inserted, `TB_CONTR_DATA.CONTR_STATUS='R'`, SQL1-SQL12 DB cleanup clears the confirmed temporary rows, physical files are not deleted by this SRS, and all changes commit atomically. |
| QA-019 | R10 | error | Submit `DISBURSEMENT_AMOUNT` greater than approved loan amount. | POST `/epl-save-isu-data-input` with `isFinish=Y`. | BE rejects or restores the amount according to page rule and does not set `EPORIS_0921=Y`. |
| QA-020 | R10 | error | Submit a collateral or purchased-property address whose front/back text exceeds 65 characters combined. | POST `/epl-save-isu-data-input` with `isFinish=Y`. | BE rejects Finished with address validation and leaves stored completion state unchanged. |
| QA-021 | R10 | error | Submit `LAW_FIRM_AMOUNT_90` without `LAW_FIRM_90_OTHER_REMARK`, an over-length remark, or an inactive law firm code. | POST `/epl-save-isu-data-input` with `isFinish=Y`. | BE rejects the request; money precision remains `NUMBER(17,2)` compatible and inactive law firm is not accepted. |
| QA-022 | R3, R11 | error | Seed valid `TB_API_AUTH` route rows but use a user who is not the current allowed case actor. | POST `/epl-case-isu-data-input-check-mb`, `/epl-case-isu-data-input-check-co`, or `/epl-save-isu-data-input`. | Service-level guard rejects the direct call; endpoint seed rows alone do not authorize the action. |
| QA-023 | R12 | happy | Seed a completed Data Input case with `TB_DISBUR_DATE.EPORIS_0921='Y'`. | POST `/epl-info-isu-data-input` and trigger Summary tab gating. | API returns `eprois0921=Y`; Summary gate proceeds using the API field while physical DB column spelling remains unchanged. |
| QA-024 | R3, R9, R11 | error | Seed valid `TB_API_AUTH` route row for return but authenticate a user who is not the current allowed actor or use a non-returnable workflow state. | POST `/epl-retu-isu-data-input`. | BE rejects with `ACCESS_DENIED` or invalid-state error; `TB_APP_HISTORY` is not inserted and workflow state is unchanged. |
