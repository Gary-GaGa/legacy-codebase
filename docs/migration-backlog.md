# 遷移清單（Migration Backlog）

> 由 D1/D2 盤點舊 JSP 後彙整。每一列 = 一個遷移任務 =
> 「複製 `deputy` 結構 + 寫 config + 接 `api.service`」對應一個 JSP 頁（見 `golden-template`）。
> 內容待 D1/D2 結果回填。

## 批次策略
- **Phase 1（垂直切片）**：挑 1 個最單純的「清單 + 表單」頁，打通 畫面→API→DB→Auth 端到端，驗證整套架構與離線環境後再規模化。
- **Phase 2+**：依**功能模組**分批；同模組一起搬（共用 model / API / 權限）。
- **排序原則**：先低風險/高重用（純清單、標準 CRUD）→ 後複雜（多步驟、報表、特殊互動）。
- **相依優先**：被多頁共用的 model / API / 共用元件先做。

## 複雜度分級（估點）
| 級 | 定義 |
|---|---|
| **S** | 純清單或單一查詢頁（直接套 `app-table-search` + config） |
| **M** | 標準 CRUD（清單 + 新增/編輯 popup + 驗證） |
| **L** | 多步驟/精靈、跨模組、報表、檔案上傳、特殊互動或大量自訂 JS |
| **XL** | 需先處理共用基礎（新共用元件、跨模組 model、難搬的 legacy 邏輯） |
