# N 軸 findings ledger — 度量驅動軸配置（斷言 → 量測）

> **動機**：`orchestration-playbook.md §4b` 的 7 軸／再審輪／軸 profile 目前是**斷言的**（「F 軸最該配資源」「採納修正後再審一輪」）。沒有數字 → 不知哪軸 ROI 高、哪軸可砍、再審輪是否值得。本 ledger 把每軸每頁的 Blocker／誤報記下，跑若干頁後用**實測 catch-rate** 回去調 §4b（軸 profile／再審輪數）。
> **權威分工**：軸定義＝`orchestration-playbook.md §4b`（本檔不重定義軸）；本檔只記「跑出來的數字」。任何據此調軸＝改 §4b（權威）+ `decisions.md` 記一行指回本檔。
> **填寫者／時機**：N 軸每跑完一頁（drain／手動）後，orchestrator 或人把**每軸一列** append 進 §1（含再審輪）。

## 3. 決策規則（達門檻才動 §4b，避免小樣本噪音）

- **樣本門檻**：累計 ≥10 頁（或 T1 ≥5 頁）才據此調軸。
- **砍軸候選**：某軸「真/頁 ≈ 0 且誤報率高」連續多頁 → 提案降為「低風險頁可省」（回 §4b 軸 profile）。**F／D 例外不可砍**（金錢/授權，最痛最空白）。
- **加碼候選**：「真/頁」最高的軸 → 提案配 2 隻不同 lens 或列不可省。
- **再審輪數**：回歸率 < 5% → 維持「再審一輪」；≥5% → 改「再審到乾淨一輪」。
- 任一調整：改 `orchestration-playbook.md §4b`（權威）+ `decisions.md` 記一行（來源指本 ledger §2）。
